# 表情包制作

## 概述

用于制作各种沙雕表情包，支持图片、文本

对于指令：

- 如果不带前缀 `/`，需要完整发送所有参数才能触发成功，不完整参数不会返回任何提醒
  - 例如 `5000生成 5000兆円 欲しい！`
- 如果带前缀 `/`，则可无需完整参数，参数不完整时会提醒您缺失的参数，在后续补全参数即可
  - 例如 先发 `/5000生成`，然后继续发送 `5000兆円 欲しい！`
  - 支持补全的同时，一次性 `/5000生成 5000兆円 欲しい！` 这样也是没问题的！
- 如果不理解上述要求，**建议直接使用**带前缀 `/` 的指令

参数中：

- 对于功能含有多段文本参数时
  - 每段文本参数之间可用 **空格** 或 **换行** 隔开
  - 每段文本参数内若有 **空格** 或 **换行**，则该段参数需要用 **半角引号** 引起来，否则可能会被认为是多个参数
    - 如 `5000生成 "我要 我要" 我还要`
  - 若每段文本参数内含半角引号，则需要用 **反斜杠** 转义
    - 如 `5000生成 他说: \"这是什么地方啊\"`
- [艾特](/about/glossary.html#%E8%89%BE%E7%89%B9) 表示使用被艾特人的头像作为**图片**
- 回复 表示使用被回复消息作为参数，不支持嵌套回复

:::tip
参数可以：

- 与指令一起发出
- 在发出指令后随后发出（需要指令前缀 `/`）
- 通过回复使用之前消息作为参数

🍥还可以三种参数形式共用
:::

如果不理解，建议直接使用带前缀 `/` 的指令，直接在下文的指令前加上 `/` 即可，例如 `/5000生成`
